# Анализатор файлов

Настольное приложение на **Python + tkinter** для детального анализа
содержимого и свойств файлов форматов **TXT, CSV, JSON**. Интерфейс
оформлен как светлая desktop-studio: просторная рабочая область, glass-панели,
монохромные Lucide-style PNG-иконки, метрики проекта и окно статистики.

## Возможности

- Загрузка TXT / CSV / JSON, автоопределение кодировки
- Создание нового проекта анализа с метаданными
- Таблица с навигацией, зеброй строк, редактированием ячеек по двойному клику
- Добавление / удаление строк и столбцов
- Фильтрация и сортировка по столбцам
- Автоматическая проверка на ошибки с точечными маркерами проблемных ячеек
- Сохранение ручных правок обратно в открытый TXT / CSV / JSON файл
- Экспорт: **CSV, JSON, Excel (xlsx), отчёт PDF, графики (PNG/PDF)**
- Окно **Статистика**: график, диаграмма и качество данных без matplotlib
- Светлая премиальная тема в стиле холодного металла / Liquid Glass
- Отдельные конфиги цветов, типографики, стилей, UI-компонентов и шрифтов

## Запуск и сборка

Стартер — основной мульти-тул проекта. Он запускает приложение, собирает
артефакт под текущую платформу и сам проверяет зависимости при каждом старте:

```bash
starter.bat          # Windows: меню запуска / сборки
./starter.command    # macOS: меню запуска / сборки
```

Команды без меню:

```bash
starter.bat run      # запустить приложение
starter.bat build    # собрать Windows x64 / Windows ARM под текущий хост
./starter.command run
./starter.command build
```

Скрипт сам:

- определяет текущую платформу и архитектуру;
- создаёт `.venv`, если его ещё нет;
- проверяет фактическое наличие модулей в `.venv`;
- при первом запуске или пропаже пакетов устанавливает `requirements.txt`;
- для сборки докачивает `pyinstaller`;
- выбирает нужный `.spec` для Windows или macOS;
- собирает артефакт в `dist/`.

Расширения, которые стартер подготавливает автоматически:

| Библиотека | Для чего |
|---|---|
| `openpyxl` | экспорт в Excel |
| `reportlab` | отчёт PDF |
| `Pillow` | экспорт графиков в PNG/PDF |

## Структура

```
app/
├── main.py            # «сборка» главного окна из mixin-ов и жизненный цикл
├── window/            # части большого класса окна, разнесённые по темам
│   ├── builders.py    # построение интерфейса (меню, тулбар, таблица, статусбар)
│   ├── table.py       # таблица: показ, фильтр, сортировка, правка, ошибки
│   ├── exporting.py   # сохранение и экспорт (CSV/JSON/Excel/PDF/график)
│   └── statistics.py  # сводки, метаданные, окно статистики
├── assets/
│   └── icons/         # локальный Lucide-style PNG icon pack
├── core/
│   └── project.py     # модель данных: загрузка, операции, валидация
├── theme/             # ВСЕ цвета, шрифты и стили — только здесь
│   ├── colors.py      # светлая палитра и токены интерфейса
│   ├── fonts.py       # типографика
│   └── styles.py      # настройка ttk
└── ui/
    ├── components.py  # glass-панели, метрики, действия с PNG-иконками
    ├── dialogs.py     # диалог создания проекта
    ├── fonts/         # пользовательские .ttf/.otf шрифты
    └── chart.py       # статистика: Canvas + Pillow, без matplotlib
scripts/
├── build_app.py       # логика стартера (запуск / сборка / зависимости)
└── strip_comments.py  # удаление комментариев и docstrings из кода
run.py                 # запуск одной командой
file_analyzer.py       # точка входа для сборки (PyInstaller)
clean_comments.bat     # очистка комментариев (Windows)
clean_comments.command # очистка комментариев (macOS / Linux)
samples/               # примерные TXT / CSV / JSON файлы
```

Класс окна большой, поэтому его методы разнесены по mixin-модулям пакета
`app/window/`, а `main.py` лишь собирает их в один класс `FileAnalyzerApp`.

Чтобы изменить внешний вид — правь файлы в `app/theme/`. Пользовательские
шрифты можно положить в `app/ui/fonts/`.

## Очистка комментариев

Чтобы быстро убрать из кода все комментарии и docstrings (например, для сдачи
«чистой» версии), запусти:

```bash
clean_comments.bat        # Windows
./clean_comments.command  # macOS / Linux
```

Скрипт безопасен: результат каждого файла проверяется компилятором Python и
перезаписывается, только если код остаётся валидным. Каталоги `.venv`, `build`,
`dist`, `__pycache__` не затрагиваются. Вернуть комментарии можно через
`git checkout -- .`.
